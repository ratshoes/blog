Security+

Section 2.4: Application Attacks

Studied injection attacks, buffer overflows, and replay attacks. SQL injection involves inserting malicious SQL queries into application inputs like web forms where developers failed to sanitize input. Buffer overflows occur when input exceeds allocated memory. For example, submitting a 10,000 character username to a field expecting 30 characters can crash legacy systems by overwriting adjacent memory. Replay attacks involve stealing valid credentials like session cookies and reusing them. An attacker captures a user's session cookie and sends it to the server, which accepts it as legitimate.
